# [VIOLATION] Assembly version reflection in BareMetalWebServer, ControlPlane, and CLI

**Severity:** 🟢 Low  
**Files:** `BareMetalWeb.Host/BareMetalWebServer.cs`, `BareMetalWeb.ControlPlane/ControlPlaneService.cs`, `BareMetalWeb.CLI/Program.cs`  
**Lines:** BareMetalWebServer.cs:138–139, ControlPlaneService.cs:~91–94, CLI/Program.cs:~assembly version read  
**Labels:** `violation`, `reflection`

## Description

Multiple entry points read the assembly version via reflection at startup:

```csharp
// BareMetalWeb.Host/BareMetalWebServer.cs:138-139
var rawVersion = typeof(BareMetalWebServer).Assembly.GetCustomAttribute<AssemblyInformationalVersionAttribute>()?.InformationalVersion
    ?? typeof(BareMetalWebServer).Assembly.GetName().Version?.ToString(3) ?? "0.0.0";
```

Similar patterns exist in `ControlPlaneService.cs` and `CLI/Program.cs`.

## Why This Violates the Guidelines

1. **Reflection** — `Assembly.GetCustomAttribute<T>()` and `Assembly.GetName()` are reflection
   APIs that read assembly metadata.
2. **Low impact** — This is a startup-only, one-time operation that reads a version string.
   It does not affect request handling or performance.

## Proposed Fix

Replace assembly reflection with a compile-time constant generated by the build system:

```csharp
// Generated by MSBuild / Directory.Build.props:
internal static partial class BuildInfo
{
    public const string Version = "1.0.0+abc1234";
}
```

Or use `AssemblyInformationalVersionAttribute` with `[assembly: ...]` in a generated file
and reference it directly via the attribute value rather than reflection.

## Affected Code Paths

- `BareMetalWebServer` constructor — reads version for display/logging
- `ControlPlaneService` — reads version for health endpoint
- CLI `Program.Main` — reads version for `--version` flag

## Resolution

**Status:** ✅ RESOLVED

All three `Assembly.GetCustomAttribute<AssemblyInformationalVersionAttribute>()` call sites have been replaced with a compile-time constant `BuildVersion.Value`. A `Directory.Build.targets` file generates a `BuildVersion.g.cs` source file per project at build time, embedding the version as an `internal const string`. The `using System.Reflection` directives have been removed from all three files (BareMetalWebServer.cs, ControlPlaneService.cs, CLI/Program.cs).
