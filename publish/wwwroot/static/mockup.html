Key Features

1. Search Box

  • Searches across all visible fields
  • Server-side filtering for performance
  • URL parameter: ?q=searchterm

2. Sortable Headers

  • Click to sort ascending/descending
  • Visual indicators (↑↓⇅)
  • URL: ?sort=Name&dir=asc

3. Pagination

  • Numbered pages with smart range
  • Shows current +/- 2 pages
  • First/Last links, ellipsis for gaps
  • Record count display

4. Page Size Selector

  • 10 / 25 / 50 / 100 items per page
  • Default: 25 items
  • URL: ?size=50

URL Examples

# Simple search /admin/data/user?q=john # Sort by email descending /admin/data/user?sort=Email&dir=desc # Page 3 with 50 items per page /admin/data/user?page=3&size=50 # Combined: search + sort + page size /admin/data/user?q=john&sort=CreatedDate&dir=desc&size=100 # Field-specific filter (backend supported) /admin/data/user?f_Email=@example.com&f_IsActive=true

Backend Field Filtering

While the UI doesn't yet include per-column filter inputs, the backend fully supports filtering via URL parameters:

# Filter by exact email /admin/data/user?f_Email=alice@example.com&op_Email=eq # Filter by age greater than 30 /admin/data/user?f_Age=30&op_Age=gt # Multiple filters (AND logic) /admin/data/user?f_Department=Engineering&f_IsActive=true # Text contains filter /admin/data/user?f_Name=John&op_Name=contains

Supported Operators

  • contains - Text contains (default for strings)
  • startswith - Text starts with
  • endswith - Text ends with
  • eq - Equals (default for numbers/dates)
  • ne - Not equals
  • gt - Greater than
  • lt - Less than
  • gte - Greater than or equal
  • lte - Less than or equal
  • in - Value in list
  • notin - Value not in list
